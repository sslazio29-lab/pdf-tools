# プロジェクト不変ルール（このファイルは改変禁止）

本ファイルはプロジェクトの統治ルールを定める。Claude Codeはこれを変更しない。
変更が必要な場合はユーザー本人のみが行う。

## ドキュメント体系
- `CLAUDE.md`（本ファイル）: 不変ルール。150行未満を維持
- `docs/SPEC.md`: 設計・仕様。編集にはユーザーの明示的許可が必要
- `docs/TASKS.md`: タスク一覧。Claudeが追加・完了更新してよい
- `docs/PROGRESS.md`: 進捗ログ。Claudeが作業のたびに必ず更新する

## 基本方針
- 変更前に必ず作業計画を提示し、承認を得る
- 一度に大量の変更をせず、小さく段階的に進める
- 不明点があれば作業を止めて質問する

## 編集権限
- `CLAUDE.md` を変更しない
- `docs/SPEC.md` はユーザーの許可なく変更しない
- `.env` を読み取り・変更しない
- テストを勝手にスキップしない

## ワークフロー
1. 計画提示 → 2. 承認 → 3. 小さく実装 → 4. 動作確認 → 5. PROGRESS.md更新
- 作業完了後は `/clear` を促す
- 長いセッションでは `/compact` を提案する

## サブエージェント活用
- 整合性チェック・監督・並列調査にサブエージェントを活用してよい
- 特に SPEC と実装の乖離チェックに用いる

## 自己成長
- 作業で得た知見（失敗含む）を PROGRESS.md に記録する
- 再利用価値のある手順は skill 化、自動化すべき事項は hook 化を提案する
- いずれも作成前にユーザーへ提案し承認を得る

## ドキュメント保守
- md群が肥大化したら最適化（要約・分割・重複削除）を提案する
- 履歴的価値の低い古い進捗はアーカイブする
