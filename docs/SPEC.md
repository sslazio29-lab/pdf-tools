# 設計・仕様（SPEC）

> このファイルの編集にはユーザーの明示的許可が必要。

## プロダクト概要
PDFの結合・分割・抽出などの操作を行えるWebアプリケーション。
どのPCからでもURLを開くだけで利用できることを重視する。

## 確定した方針（議論済み）
- **形態**: Webアプリ（インストール不要、URLで利用）
- **処理場所**: 原則ブラウザ内で完結。機密PDFを外部に送らない
- **ホスティング**: 当面は GitHub Pages（無料・静的配信）
- **サーバー処理**: 現時点では不要。重い処理が必要になった場合のみ将来検討（例: Cloudflare Workers）

## 想定機能
### 第1段階（中核）※実装済み
- 結合: 複数PDFを1つにまとめる
- 分割・抽出: ページ範囲で切り出し、特定ページの抽出

### 第2段階
- ページ編集: 並べ替え・削除・回転 ※実装済み
- 圧縮: ファイルサイズ削減 ※恩恵が少ないためスキップ（ユーザー判断 2026-06-16）

### 追加実装済み
- OCR: スキャンPDFの文字認識（ブラウザ内、Tesseract.js、エンジン/言語データは実行時CDN取得）。検索可能PDF（透明テキスト層）の生成にも対応
- 変換: 画像 ⇔ PDF（画像→1PDF、PDF→画像ZIP）

### 将来候補（未着手）
- 透かし・ページ番号付与

## 技術スタック（暫定）
- ビルド: Vite
- UI: React + TypeScript
- PDF操作: pdf-lib
- PDF表示（サムネイル）: pdf.js
- OCR（将来）: Tesseract.js（WebAssembly、遅延読込）

## 制約・割り切り
- 既存テキストの直接編集は当面対象外（難易度高）
- 巨大ファイル（数百MB）はブラウザのメモリ制約あり
- OCRの言語データは1言語10〜15MB。利用時のみダウンロード

## サイズ目標
- OCRを使わない通常利用時の初回読み込み: 2MB前後を目安

## 確定済み（旧・未確定事項）
- UIフレームワーク: React + TypeScript（確定）
- レイアウト: 機能ごとのタブ式UI（結合／分割・抽出／ページ編集／OCR／画像⇔PDF変換）
- 対応ブラウザ: WebAssembly対応のモダンブラウザ（Chrome/Edge/Firefox等の最新版）を想定
